技術趨勢 Trends

01

代理執行時（Agent Runtime）成為核心基礎設施競逐點

LangChain 發布《Generative Agent Runtime》深度指南（24 分鐘閱讀），CrewAI CEO 宣告「Harness 已死」——兩大框架同步指向同一個訊號：業界正從「如何串接工具」轉向「如何設計穩定、可觀測、可擴展的代理執行環境」。Runtime 層的競爭正式開打。

02

文件解析基準化：ParseBench × Kaggle 開放競技場

LlamaIndex 與 Kaggle 合作推出業界首個文件 OCR 公開排行榜，涵蓋表格、圖表、格式保留、忠實度、視覺接地 5 大維度。這是 AI Agent 工作流中「資料輸入品質」從主觀評測走向客觀基準的里程碑，將加速競品收斂到可比較的精度標準。

03

後台子代理（Background Subagent）架構普及

LangChain 的《Running Subagents in the Background》和 Anthropic 的多代理 SDK 同步強調異步子代理設計。代理任務的「主流程不阻塞 + 背景並行 + 結果彙整」架構正在成為生產部署標準模式，對複雜工作流的吞吐量有 3-10 倍提升潛力。

04

Agent 評估與可觀測性工具趨向企業標準化

LangSmith 推出可重用評估器範本、CrewAI 推出 AMP 管理平台、Anthropic 強化 Claude 多步驟任務追蹤——三大生態同步往「讓企業可衡量、可比較、可審計 Agent 行為」方向聚攏，顯示評估工具已從研究側進入產品側。

05

AI 安全從口號落地為合約與聯盟

Anthropic 主導的 Project Glasswing 擴展至 12 家機構（含 Amazon、Apple、Cisco、Google、NVIDIA），並確認 NSA 已部署 Claude Mythos 資安專用模型。AI 安全正從學術研究轉為有法律約束力的產業聯盟行為，對 Agent 部署合規性要求將隨之提升。

深度分析 Deep Analysis

Agent 執行層：從框架競爭到基礎設施競爭

2026 年 4 月，AI Agent 領域正在經歷一次悄然但深刻的結構性轉變：競爭焦點從「哪個框架 API 更好用」轉移到「誰能提供更穩定、可觀測、可擴展的 Agent 執行環境」。

LangChain 的《Generative Agent Runtime》深度指南，代表這一轉型最清晰的信號。24 分鐘的閱讀量說明這不是一篇工具教學，而是一份生產部署架構指引——涵蓋狀態管理、錯誤恢復、工具呼叫生命週期、並發控制等生產環境必須面對的複雜問題。這些問題在早期 Agent 概念驗證階段可以忽略，但要上線服務千人以上的規模，每一個都是必須回答的工程問題。

CrewAI CEO João Moura 的《Agent Harnesses are Dead》則從另一個角度印證了同樣的趨勢。他指出舊式 harness（以腳本、胡亂串接為特徵的代理架構）面臨根本性挑戰：缺乏記憶持久化、缺乏狀態追蹤、缺乏自我修復能力。他的「Long Live Agent Harnesses」暗示新一代 harness 需要原生支援認知記憶與自我演化，而非只是工具的容器。

在評估與可觀測性維度，LangSmith 可重用評估器範本的推出，以及 LlamaIndex 與 Kaggle 合作的 ParseBench OCR 排行榜，共同指向 Agent 工作流品質管理的系統化。尤其後者意義深遠：文件解析是幾乎所有知識型 Agent 的輸入層，解析精度的可量化、可比較，將讓「用 AI 讀文件」從玄學變成可工程化的問題。

Anthropic 在這個時間點推出 Claude Opus 4.7，同時強化 Project Glasswing 的產業聯盟，說明頂層模型廠商的策略是：用更強的基礎模型佔據 Agent 的推理核心，同時以安全聯盟建立合規部署的信任背書。Claude Mythos 被 NSA 採用，是這一策略在最敏感場景的落地驗證。

綜合四個來源的訊號，2026 年 Q2 的 AI Agent 發展主軸可以概括為：執行時架構成熟化、評估工具標準化、安全部署合規化。開發者若要在這一波浪潮中保持領先，需要開始認真對待 Agent 的 DevOps 層——不只是「讓 Agent 能跑」，而是「讓 Agent 能穩定、可追蹤地跑在生產環境」。

建議行動 Recommended Actions

建議 01

評估並試用 LangChain Generative Agent Runtime 架構

若你的 Agent 系統已進入或即將進入生產環境，《Generative Agent Runtime》是本週最值得精讀的技術文章。重點關注：狀態管理策略、工具呼叫的錯誤恢復機制、並發子代理的協調模式。可以對照自己目前的 Agent 架構做差距分析。

建議 02

在 ParseBench 排行榜上測試你的文件解析方案

LlamaIndex × Kaggle 的公開排行榜提供了一個難得的客觀基準。如果你的 Agent 工作流有文件輸入環節，建議把現用的解析方案（無論是 LlamaParse、自建 OCR、還是其他方案）拿到 ParseBench 跑一次，建立基線數據。這將是後續優化的起點。

建議 03

升級評估框架：從一次性測試走向可重用評估器

LangSmith 可重用評估器範本的推出，是個好機會重新審視你的 Agent 評估流程。如果目前是手動抽測或一次性腳本，考慮將評估邏輯封裝為標準化範本，便於不同模型版本間的橫向比較，以及新功能上線前的自動化回歸測試。

建議 04

關注 Claude Opus 4.7 在多步驟 Agent 場景的表現

Anthropic 特別強調 Opus 4.7 在 multi-step tasks 上的性能提升。如果你目前使用 Claude 系列模型，建議在關鍵的多步驟工作流場景進行 A/B 測試（4.5 vs 4.7），量化精度提升是否足以抵消成本增加。

建議 05

評估 Agent 部署的安全合規要求

Project Glasswing 的 12 家機構聯盟、NSA 採用 Claude Mythos，預示企業級 Agent 部署的合規門檻將提高。建議現在開始整理你的 Agent 系統：工具呼叫是否有審計日誌？敏感資料是否有存取控制？輸出內容是否有人工審核機制？提前建立文件，避免未來合規壓力突襲。

專有名詞解釋 Glossary

Agent Runtime
代理執行時環境。負責管理 AI Agent 的執行生命週期，包含狀態管理、工具呼叫調度、錯誤恢復、並發控制等基礎設施層。類比軟體開發中的「執行時（Runtime）」概念。
Agent Harness
代理框架骨架。指用來串接、協調多個 AI 代理的腳手架程式碼或框架。CrewAI CEO 用「Harness is Dead」隱喻舊式腳本型框架已被新一代有狀態、有記憶的代理系統取代。
Background Subagent
後台子代理。在主代理流程之外非同步執行的子任務代理，結果完成後再回傳給主流程。可大幅提升複雜工作流的吞吐量，避免主流程等待。
ParseBench
文件解析基準。LlamaIndex 推出的 AI 代理文件解析評測框架，包含約 2,000 個人工驗證企業文件頁面，涵蓋表格、圖表、格式保留、忠實度、視覺接地 5 大維度。
LiteParse
LlamaIndex 推出的輕量級開源本地文件解析器，使用網格投影演算法從 PDF 提取文字，無需 Python 依賴即可在本地執行，已獲 4,300 GitHub 星標。
Deep Extraction
深度提取架構。相對於「單次提取（Single-Pass Extraction）」的多輪驗證文件提取框架，內建問責迴圈，可標記遺漏內容並觸發補充提取，確保生產環境資料完整性。
LangSmith Fleet
LangChain 的代理群組管理與可觀測性平台，提供多代理系統的統一監控、追蹤、評估功能，現已支援整合外部工具（如 Arcade.dev）的追蹤。
Project Glasswing
Anthropic 主導的軟體供應鏈安全聯合計畫，12 家科技機構參與，利用 AI 輔助發現和修補關鍵軟體漏洞，目標建立業界級 AI 資安防護標準。
Claude Mythos
Anthropic 的資安專用 Claude 模型，特別優化滲透測試、威脅分析等資安場景，已被美國 NSA（國家安全局）採用部署。
MCP（Model Context Protocol）
模型上下文協定。Anthropic 開發並已捐贈給 Agentic AI Foundation 的開放標準，定義 AI 模型與外部工具、資料來源之間的通訊介面，使不同廠商的 Agent 工具可互通。
CrewAI AMP
Agent Management Platform，CrewAI 的企業代理管理平台，分三個層次：AMP Cloud（雲端視覺化管理）、AMP Factory（企業內部部署）、OSS（開源編排框架）。
VLM（Vision Language Model）
視覺語言模型。能同時處理圖像與文字的多模態大型語言模型，常用於 OCR、文件理解、圖像問答等場景。LlamaIndex 文章中討論其在生產 OCR 系統中的故障模式。
