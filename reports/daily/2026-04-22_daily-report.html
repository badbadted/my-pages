五、深度分析

「Agent Harnesses are Dead」的深層含義

João Moura 的文章標題看似聳動，但背後邏輯清晰：早期 Agent 框架（包括 LangChain 的舊版 Executor、甚至 CrewAI 自身早期版本）以「harness」為核心——預設流程、硬編碼工具清單、固定角色綁定。2026 年的代理系統正在轉向自主任務規劃 + 動態工具選擇 + 記憶持久化三位一體的架構。LangChain 的 Deep Agents 和 CrewAI 的 AMP 都朝此方向演進，Anthropic Claude Agent SDK 亦然。

Amazon $50 億對 Claude Agent SDK 的影響

這不僅是財務事件。Amazon 的條件是 Anthropic 承諾 $1,000 億 AWS 雲端支出，意味著 Claude 的推論基礎設施將深度鎖定在 AWS Trainium/Inferentia 晶片上。對開發者而言，未來 Claude Agent SDK 的最佳部署路徑大概率是 AWS Bedrock，而非其他雲端平台。

文件解析 AI 的基礎設施化

ParseBench 的推出（業界首個客觀基準）預示文件解析正在從差異化功能走向標準化基礎設施。當評估標準確立，競爭者將快速湧入，LlamaIndex 的先發優勢是 ParseBench 自身——它既是標準又是廣告。LiteParse 4,300 ⭐ 的快速增長印證了開源社群對本地解析能力的強烈需求。

八、今日學習重點

🧠 從「工具清單」到「自主規劃」

CrewAI 的「harness is dead」宣言提醒我們：真正的代理不是預定義工具的執行者，而是能夠動態規劃、選擇工具、調整策略的系統。設計代理時，先問「它需要什麼工具」不如先問「它需要完成什麼目標」。

🧠 文件解析是 RAG 的隱形瓶頸

LlamaIndex 的系列文章（ParseBench、Deep Extraction、OCR 分析）統一指向一個洞察：大多數 RAG 系統的真正問題不在模型，而在資料解析品質。「垃圾進、垃圾出」在文件 AI 尤其明顯。

🧠 資安模型的分化趨勢

Claude Mythos 被 NSA 採用而 Sam Altman 批評其為「恐懼行銷」，顯示 AI 安全模型市場正在分化：一派主張特化資安模型（Anthropic Mythos），另一派主張通用模型的安全外掛（OpenAI 方向）。兩種路線各有使用場景。
