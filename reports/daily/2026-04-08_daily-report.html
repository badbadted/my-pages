🎓 學習重點

學習一：Sub-agent 架構的設計原則

Deep Agents v0.5 的子代理架構提出了一個重要的設計問題： 何時應該讓主 Agent 分裂出子代理，而不是直接處理任務？ 核心判斷依據是任務的「獨立性」——如果子任務可以並行執行且結果互不依賴， 子代理架構才能發揮效益。強行用子代理處理串行依賴任務反而增加通訊開銷。

學習二：AI 安全從研究議題轉為工程議題

Project Glasswing 代表 AI 安全的一個重要轉折：從「學術研究安全風險」轉為「工程部署安全防護」。 對 Agent 開發者來說，這意味著未來的 Agent 部署清單中， 「安全合規性驗證」將成為與「功能正確性驗證」同等重要的要求。 現在開始建立安全測試框架比事後補救更有效率。

學習三：文件處理的「推理優先」思維轉換

LlamaIndex 的 Agentic OCR 框架核心洞察是：傳統 OCR 把文件處理視為解析問題（規則驅動）， Agentic OCR 把它視為推理問題（模型驅動）。 這個思維轉換適用於所有「非結構化資料處理」場景—— 當資料格式複雜多變時，讓 Agent 推理如何提取往往比硬編碼規則更穩健。
