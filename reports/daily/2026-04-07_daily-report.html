深度分析 Deep Analysis

主題一：「Agentic OCR」— 文件 AI 的典範轉移

LlamaIndex 今日大規模發布 12 篇 Agentic OCR 系列文章，這不只是內容行銷策略，更是一個技術典範轉移的宣示。傳統 OCR（光學字元辨識）是靜態的規則式系統，面對低品質掃描、複雜佈局、多語言混排時容易失效。

Agentic OCR 的核心差異：

  • 推理能力：Agent 能理解文件語義，而非僅提取字元序列
  • 自我修正：偵測提取錯誤並自動重試，無需人工介入
  • 佈局理解：理解表格、圖表、多欄排版的視覺幾何結構
  • 端到端整合：直接連接下游系統（ERP、CRM、資料庫），而非輸出靜態文字

LlamaParse 號稱 Agentic OCR 直通率達 90-95%+，相較傳統 OCR 30-50% 的人工審核率，這是量級上的提升。

主題二：LangChain 的「Agent 持續學習三層架構」

Harrison Chase 在最新文章中提出 AI Agent 的學習不應只看「訓練」，而應從三個層面理解：

  • 第一層：模型權重更新（最重）— 傳統微調，成本高、週期長
  • 第二層：上下文內學習（中）— Few-shot、RAG 注入，即時但受 Context Window 限制
  • 第三層：外部記憶持久化（輕）— 長期知識庫，跨 Session 累積，可版本管理

此框架對 Agent 系統設計有直接意義：生產級 Agent 應同時善用三層，根據任務性質決定學習路徑。這也與 CrewAI 的 Cognitive Memory 研究遙相呼應，表明「Agent 記憶管理」已成為業界共識的下一個關鍵問題。

術語解釋 Glossary
Agentic OCR
結合 AI Agent 推理能力的光學字元辨識。與傳統 OCR 不同，Agentic OCR 能自主推理文件結構、偵測並修正錯誤、處理複雜佈局，不依賴固定規則。
Agent Harness
LangChain 提出的 Agent 執行框架架構，定義 Agent 如何與工具（Tools）、記憶（Memory）和外部環境互動的標準結構層。
LangSmith Fleet
LangChain 的企業 Agent 車隊管理平台，支援多 Agent 部署、Skills 共享，以及 Assistants / Claws 兩種授權模型，用於大規模 Agent 生產管理。
Continual Learning（持續學習）
Agent 在部署後持續學習並改善的能力。Harrison Chase 的框架定義三層：模型微調、上下文內學習、外部記憶持久化，對應不同成本與即時性需求。
Self-Healing Agent（自愈 Agent）
能自動偵測執行錯誤或迴歸、診斷根因並產生修復方案的 Agent。LangChain GTM Agent 實作了自動偵測→診斷→開 PR 修復的完整迴路。
Cognitive Memory（認知記憶）
CrewAI 提出的 Agentic 記憶架構，模擬人類認知的分層記憶機制（工作記憶、短期記憶、長期記憶），解決 Agent 跨任務的上下文管理問題。
LiteParse
LlamaIndex 的開源本地文件解析器，零 Python 依賴，可在本地端執行，適合隱私敏感或離線場景。是 LlamaParse 雲端版的輕量替代方案。
NemoClaw（NVIDIA）
NVIDIA 的 Agent 加速框架，與 CrewAI 整合後實現 100x 工作流速度提升，適合計算密集型多 Agent 並行執行場景。
Responsible Scaling Policy（RSP）
Anthropic 的內部 AI 安全擴展政策，第 3 版（v3.0）定義在不同能力門檻下的安全評估要求與部署限制，是業界重要的 AI 治理參考框架。
MCP（Model Context Protocol）
Anthropic 提出的模型上下文協議，定義 AI 模型如何與外部工具、資料來源標準化互動。目前已被多個 AI 框架採用作為工具整合標準。
LlamaAgents Builder
LlamaIndex 的自然語言 Agent 創建平台，用戶可用文字描述需求，系統自動生成對應的 Agent 工作流並一鍵部署，降低 Agent 開發門檻。
CrewAI AMP
Agent Management Platform，CrewAI 的企業 Agent 管理平台，提供 Agent 監控、調度、版本管理功能，搭配 AMP Cloud 與 AMP Factory 形成完整企業部署路徑。
