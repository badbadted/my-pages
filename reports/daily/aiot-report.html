Zigbee2MQTT 升級至 2.11.0 — 安全強化：升級前先確認外部轉換器設定
⏱ 30 分鐘
2.11.0（6/1）把 external extensions 與 external converters 改為預設停用，避免載入未經驗證的外部 JS 程式碼——方向正確的資安強化。但若你目前靠外部轉換器支援某些非標準裝置（常見於部分 Tuya / 客製感測器），升級後它們會無聲失效，必須手動在設定中重新開啟。另含 Zigbee Direct cluster 支援、GreenPower 解析強化、HA discovery Jinja-safe 修正，及約 18 款新裝置。
問題 / 機會
資安預設值收緊是好事，但「預設停用」對既有安裝是潛在破壞性變更——盲升可能讓部分裝置從 HA 消失。升級前先盤點是否有 external converter。
建議做法
先檢查 Z2M 設定有無 external_converters / external_extensions，有就在升級後重新啟用白名單。
🆕 本期新增 2.11.0（6/1） ⚠️ 外部轉換器預設停用，需手動開啟 官方 changelog 頁持續 404，以 GitHub Releases 為準 📋 GitHub Releases →
操作步驟
  1. 升級前檢查 configuration.yaml 是否有 external_converters: 或 external_extensions: 區塊，記下清單
  2. HA Add-on Store 升級，或 docker pull koenkk/zigbee2mqtt:latest && docker restart z2m
  3. 升級後若上述清單非空，於設定中重新開啟對應外部轉換器/擴充並重啟
  4. 觸發各裝置開/關，確認 HA 中所有 Zigbee 實體仍在線、無消失
🟡 建議先測試 · 下次維護視窗執行 相關服務：192.168.0.222 Zigbee2MQTT
升級 Home Assistant Core 至 2026.5 — 維護儀表板 + RF 射頻支援（你落後兩版）
⏱ 1–2 小時
你的 HA Container 停在 2026.3.1，官方最新仍是 2026.5（5/6）（6/3 本次重抓無更新發行版）。最實用的是 維護儀表板（Maintenance Dashboard）：一畫面列出所有低電量裝置 —— 對你 178 個 FP2 實體 + Zigbee 電池裝置群的維運直接有感。另有 RF 射頻裝置支援（捲門/車庫門/門鈴可入 HA）與 ESPHome 序列埠網路化。
問題 / 機會
落後兩版意味著錯過維護儀表板、12 項新整合與自動化編輯器改善；版本差距越大，未來一次跳升的整合破壞風險越高。
建議做法
分兩跳升級 2026.3.1 → 2026.4（IR 一級公民）→ 2026.5，每跳後驗證整合。
沿用上期（仍最新發行版） 2026.5（5/6） ⚠️ 跨兩版，先備份再升 📋 HA Blog →
操作步驟
  1. 備份：HA → Settings → System → Backups → 立即建立完整備份
  2. 先升 2026.4：docker pull homeassistant/home-assistant:2026.4 && docker restart homeassistant，驗證 FP2 / Z2M / ESPHome / WLED 整合無紅字
  3. 再升 2026.5：改 tag 為 :stable 重啟，開啟維護儀表板確認低電量裝置清單正常
  4. 逐一檢查核心整合實體狀態與既有自動化（迎賓 TTS / 自動關冷氣）仍運作
🟡 建議先測試 · 下次維護視窗執行 相關服務：192.168.0.222 HA Container (Docker/OrbStack)
ESPHome 升級至 2026.5.0 — 省電模式電流近半 + Device Builder 新介面
⏱ 1–2 小時
主迴圈與排程器架構重大改進，省電模式電流從 2.0 mA 降至 1.1 mA（電池供電 ESP 裝置續航直接受益）；修正 watchdog feed 率 1000× 回歸。另推全新 Device Builder Web 介面（視覺化元件 + 韌體排程佇列）取代舊 Dashboard，新增 ESP32-C6 / H2 Zigbee 曝露、音訊堆疊省 40% CPU。
問題 / 機會
你的 ESPHome 裝置群升級後待機功耗與 CPU 開銷明顯下降；C6/H2 Zigbee 支援讓未來 ESP32 直接走 Zigbee 不必額外橋接。
沿用上期（仍最新） 2026.5.0 ⚠️ modbus_controller server 模式 → 改 modbus_server 📋 ESPHome Changelog →
操作步驟
  1. HA Add-on（ESPHome Device Builder）更新至 2026.5.0
  2. 若有用 modbus_controller server 模式，先改成新的 modbus_server 元件再重編
  3. 逐台 OTA 重編韌體；電池裝置確認省電模式生效後再批次推
🟡 建議先測試 · 單台驗證後批次推 相關服務：192.168.0.222 HA + ESPHome 裝置群
Frigate 升級至 0.17.1 — 安全修復：限制 raw config 端點 + 修正跨攝影機鑑權
⏱ 1 小時
0.17.1 為安全維護版：raw config 端點限定 admin 角色、修正 timeline / media 端點的跨攝影機鑑權漏洞、exec/expr/echo go2rtc 來源預設停用。你的 Frigate NVR 對外暴露時這兩項鑑權修復值得儘快套用。
問題 / 機會
跨攝影機鑑權漏洞可能讓低權限使用者越權存取其他攝影機畫面；屬可儘快處理的安全項。
沿用上期（仍最新） 安全相關 ⚠️ 若從 0.16 升，genai 設定需遷移至 objects → genai 📋 GitHub Releases →
操作步驟
  1. 備份 Frigate config；若仍在 0.16，先把 genai 區塊遷移到 objects → genai
  2. 升級容器並重啟，確認攝影機串流與物件偵測正常
  3. 驗證低權限帳號無法存取 raw config 與其他攝影機端點
🟡 安全項 · 建議近期執行 相關服務：Frigate NVR + RTX 5070 Ti（GenAI 加速）
