P2 — 精度優化

模糊比對 + 音素相似度 P2 中優先

Whisper 有時會把 "Hey Luna" 辨識為 "Hey Lunar"、"Helena"、"Hey Luma" 等。 用模糊比對可以提高召回率。

# 比對策略
exact: "luna", "hey luna"
fuzzy: "luma", "lunar", "loona", "runa"
phonetic: Soundex/Metaphone 演算法
threshold: Levenshtein distance ≤ 2
openWakeWord CH1 重訓 P2 中優先

之前訓練失敗是因為用了 CH0（DSP 通道）。現在改用 CH1（ASR 輸出）後音質正確， 應能成功訓練 openWakeWord 模型。

  1. 用 CH1 錄 50+ 次 "Hey Luna"（不同距離、音量、語氣）
  2. 用 CH1 錄 10+ 分鐘環境噪音 + 日常對話
  3. 提取 openWakeWord embedding → 訓練分類器
  4. 部署為雙層架構的第一層
dscripka/openWakeWord — Colab 訓練 notebook
Speex 降噪整合 P3 低優先

openWakeWord 原生支援 Speex 降噪。在送 VAD/喚醒詞偵測前先過 Speex， 可提升吵雜環境（客廳電視開著）的辨識率。XVF3800 CH1 已有 beamforming 但沒有 NS。

P2 — 語音 UX 提升

對話模式（Conversation Mode） P2 中優先

喚醒後保持 5-10 秒的「活躍監聽」狀態，不用每次都說 "Hey Luna"。 例如：

你：Hey Luna，現在幾度？
露娜：現在 28 度。
你：開冷氣（不用再說 Hey Luna）
露娜：冷氣已開，設定 26 度。
（5 秒無聲 → 回到待機）
插話偵測（Barge-in） P2 中優先

露娜在 HomePod 播報時，用戶可以打斷。需要 AEC（回聲消除）濾掉 HomePod 的聲音， 才能在播報中聽到用戶的聲音。XVF3800 內建 AEC 但需要 reference signal 設定。

聲紋辨識（Speaker ID） P3 低優先

辨識是誰在說話，提供個人化回應（爸爸 vs 小孩）。 可用 pyannote-audio 或 resemblyzer 做聲紋嵌入比對。

pyannote/pyannote-audio — 說話者分離 + 辨識

P3 — 硬體擴展

Atom Echo 喚醒整合 P3 低優先

專用喚醒晶片（Syntiant NDP） P3 低優先

Syntiant NDP 是專門為 always-on 語音偵測設計的神經處理器，功耗 <150μW。 可做成獨立的喚醒模組，喚醒後才啟動主系統。但需要額外硬體，暫不急。

建議執行順序

1
對話模式
喚醒後保持 5-10 秒活躍監聽，不用重複說 Hey Luna。改動小，UX 提升大。
2
模糊比對強化
加入 Soundex/Levenshtein 比對，提高喚醒詞辨識容錯率。純軟體改動。
3
whisper.cpp + CoreML
替換 faster-whisper，用 Apple Silicon 硬體加速，延遲降 3-5 倍。
4
SenseVoice 評估
測試中英混合辨識精度，如果比 Whisper 好就替換完整 STT 引擎。
5
openWakeWord CH1 重訓 + 雙層架構
用正確的 ASR 通道收集樣本，訓練 openWakeWord 做第一層快篩，降低 CPU 消耗。
6
Atom Echo 多房間部署
臥室、書房各放一個 Atom Echo，microWakeWord 本地喚醒 → 串流到 Mac STT。
