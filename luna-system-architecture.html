2. 主要服務

  • OpenClaw Gateway — LINE webhook 接收、agent 調度
  • Voice Bridge — 語音/設備控制 API、Web 前端存取
  • Luna Server — 網頁 Chat（SSE 串流）、內部 API
  • ReSpeaker Listener — 喚醒詞控制 + 按鈕觸發
  • Home Assistant（OrbStack VM）— 設備控制
  • SoVITS TTS（Docker）— 文字轉語音
  • Mosquitto MQTT — Zigbee2MQTT 通訊

9. 容錯降級機制

// Luna Server 內部降級 主力模型: gemini-2.5-flash-lite ↓ 失敗 2 次 備用模型: gemini-2.5-flash ↓ 失敗 5 次 → Circuit Breaker 開啟（60s 冷卻） Luna 完全掛掉 // Voice Bridge 外部降級 Luna 無回應（120s 超時） ↓ fallback: ask_openclaw() → Kimi K2.5 ↓ 通知: _notify_luna_down() → LINE 推送告警 // 60 秒去重，不會狂發通知

10. 完整範例：「開冷氣 26 度」

① 用戶對 ReSpeaker 說「Hey Luna，開冷氣 26 度」 ReSpeaker CH1 → Silero VAD 偵測語音 ↓ 收集 1.2 秒語音 → 存 WAV ↓ Whisper tiny (英文) → "hey luna" → ✓ 包含 "luna" ↓ 🔔 播放 chime 提示音 ↓ 錄音 3.5 秒（偵測到靜音自動停止） ↓ Whisper base (中文) → "開冷氣26度" ↓ POST Voice Bridge /say → 秒回 200 ↓ 背景非同步 POST Luna /api/voice {"q":"開冷氣26度", "src":"respeaker"} ↓ Gemini Function Calling: → 識別意圖: control_ac → 參數: action="on", temp=26, mode="cool" ↓ executeTool('control_ac') → POST Voice Bridge /ac → HA SmartIR → 紅外線發射 → 冷氣啟動 ↓ Gemini 回覆: "冷氣已開啟，設定 26 度 ❄️" ↓ speak_tts → SoVITS (嘉嵐) → ffmpeg → HomePod RAOP ↓ 🔊 HomePod 播報：「冷氣已開啟，設定 26 度」
